## 详细风险评估结果

### 1. LRU 缓存策略 - 🟡 中风险 ✅ 推荐实施

**关键风险**:
- Map 迭代器分配开销（~50μs per eviction）
- 批量淘汰可能产生 1.6ms 尖峰（32 buckets 同时淘汰）
- WeakMap 清理延迟可能积累 8MB 内存

**缓解措施**:
- 使用迭代器避免 Array.from 分配
- 淘汰频率监控，>1000/s 时自动降级
- Feature flag: `VUE_TUI_LRU_CACHE`

**回退方案**: ✅ 必需

---

### 2. 对象池 - 风险分化

#### ✅ 字符串构建器池 - 🟡 中风险 - 推荐实施

**风险**: 生命周期管理、状态污染  
**缓解**: try-finally 保护 + 强制清理

#### ✅ Map 对象池 - 🟡 中风险 - 推荐实施

**风险**: 嵌套对象未清理、池满降级  
**缓解**: 清理断言 + 自适应池大小

#### 🔴 Span DOM 元素池 - 🔴 高风险 - 暂缓实施

**风险**: 事件监听器残留、CSS 污染、DOM 泄漏  
**建议**: 暂缓，需要更完善的清理机制

---

### 3. ASCII 快速路径 - 风险分化

#### ✅ 完整 ASCII 检测 - 🟢 低风险 - 强烈推荐

**收益**: 5x 加速  
**风险**: ASCII 控制字符处理需要增强  
**缓解**: 在 isAscii 中排除 \t, \r, \n

#### 🔴 启发式优化 - 🔴 高风险 - 不推荐

**风险**: 假阳性率 5-15%，导致宽度计算错误  
**触发**: "Hello😀World" 误判为 ASCII  
**建议**: 不实施

---

### 4. 临时数组复用 - 🟢 低风险 ✅ 强烈推荐

**风险评估**:
- Slice 视图被修改: 🟢 低（Object.freeze 保护）
- 并发竞态: 🟢 低（单线程同步）
- 内存碎片: 🟢 低（<10KB）

**收益**: -30% GC 压力

---

### 5. 缓存大小调整 - 🟡 中风险 ⚠️ 有条件推荐

**内存影响**: +6.3 MB（最坏情况）

**环境评估**:
- 桌面: ✅ 可忽略（<1%）
- 移动端: ⚠️ 需要降级（1-6%）
- 嵌入式: 🔴 显著（6-12%）

**缓解**: 环境自适应

```typescript
const MAX_SIZE = globalThis.navigator?.deviceMemory <= 4 ? 128 : 256;
```

## 关键发现

### 🚨 必须暂缓的优化

1. **Span DOM 元素池（P1-15）**
   - 原因: DOM 清理复杂，容易引入 subtle bugs
   - 风险: 事件监听器泄漏、样式污染
   - 建议: 需要重新设计清理机制

2. **ASCII 启发式优化（P1-14）**
   - 原因: 假阳性率 5-15% 太高
   - 风险: 宽度计算错误 → 文本截断
   - 建议: 使用完整 ASCII 检测
